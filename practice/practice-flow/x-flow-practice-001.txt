//Watermarking use case:
In this scenario, the existing application continues inserting new records into the database until the
environment is fully retired and the new salesforce-based solution goes live. To handle this, we use 
watermarking to ensure that only incremental data—records created after the previous run—are fetched 
and processed. first, we perform an initial full data migration from the legacy database to Salesforce. after 
that, mule captures and stores a watermark value, typically the last-processed timestamp or Id, and uses 
it in scheduled jobs to continuously migrate only newly inserted records. mule performs one last incremental 
sync based on the watermark, ensuring no data loss and a seamless migration to salesforce..
Always use ✅ persistent for water-marking

//Consolidating (combine) two payload into single response logic:

Db Select Query: select emp_id , emp_name ,emp_status from onboard.emp_master

Set variable Name: empDbPayload , value: payload

HTTP Request URL: http://empfinance-api-cloudhub.io/employees

Transform Message: 
%dw 2.0
output application/json
---
(payload map(item , index) ->{
(vars.empDbPayload filter($.emp_id == item.id) map{
"id": $.emp_id ,
"name": $.emp_name ,
"status": $.emp_status ,
"salary": item.emp_salary ,
"age": item.emp_age
})
}) filter($ != {})

This DataWeave transformation merges two arrays of employee data by iterating over the main payload using map, 
and for each item it iterates again over the empDbPayload variable to find the matching record. Inside the 
inner loop, the $ represents the current object from empDbPayload, while item refers to the current object from 
the outer iteration. The expression filter($.emp_id == item.id) ensures that the rows are matching  between 
two array of object payloads.. For every matched pair, the map builds a consolidated employee object by taking 
fields like id, name and status from empDbPayload and combining them with additional fields like salary and age 
from the main payload. Finally, the outer filter($ != {}) removes any empty results created when no matching 
record was found. this is a common DataWeave pattern used to merge two arrays based on a matching field and 
produce a single combined response...

//Handling guaranteed-delivery in jms with objectStore and activeMQ: jitendra-bapana


		name="Publish-JMS-Payload"
		<http:listener
			doc:name="Listener"
			config-ref="HTTP_Listener_config"
			path="/jms"
			allowedMethods="POST" />
		<jms:publish
			doc:name="Publish"
			destination="testQueue"
			config-ref="JMS_Config">
			<jms:message outboundContentType="application/json" />
		</jms:publish>
	</flow>

	<flow
		name="Get-JMS-Payload-Invloke-HTTP-Request"
		<jms:listener
			doc:name="On New Message"
			config-ref="JMS_Config"
			inboundContentType="application/json"
			destination="testQueue">
			<jms:response outboundContentType="application/json" />
		</jms:listener>
		<try
			doc:name="Try"
			<until-successful
				maxRetries="3"
				doc:name="Until Successful"
				<http:request
					doc:name="Request"
					method="POST"
					url="http://localhost:8081/process" />
			</until-successful>
			<error-handler>
				<on-error-propagate
					enableNotifications="true"
					logException="true"
					doc:name="On Error Propagate"
					type="ANY">
					<os:store
						doc:name="Store"
						key='#[now() as String{format:"yyyyMMddHHMMSS"}]'
						objectStore="Object_store" />
				</on-error-propagate>
			</error-handler>
		</try>
	</flow>

	<flow
		name="Log-JMS-Payload"
		<http:listener
			doc:name="Listener"
			config-ref="HTTP_Listener_config"
			path="/process" />
		<logger
			level="INFO"
			doc:name="Logger"
			message="--------------#['\n'] #[payload] #['\n']\------------------------" />
	</flow>

	<flow
		name="Collect-Fail-JMS-Payload"
		<http:listener
			doc:name="Listener"
			config-ref="HTTP_Listener_config"
			path="/reprocess" />
		<os:retrieve-all-keys
			doc:name="Retrieve all keys"
			objectStore="Object_store" />
		<foreach
			doc:name="For Each"
			<set-variable
				value="#[payload]"
				doc:name="Set Variable"
				variableName="vKey" />
			<os:retrieve
				doc:name="Retrieve"
				key="#[vars.vKey]"
				objectStore="Object_store" />
			<logger
				level="INFO"
				doc:name="Logger"
				message="#[payload]" />
			<os:remove
				doc:name="Remove"
				objectStore="Object_store"
				key="#[vars.vKey]" />
		</foreach>
		<logger
			level="INFO"
			doc:name="Logger"
			message="---------------#['\n'] processing is completed #['\n'] ---------------" />
	</flow>
