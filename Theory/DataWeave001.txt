DataWeave is a 'functional-programming' language used in muleSoft for data-transformation. It allows developers 
to convert data from one format to another, such as JSON to XML, CSV to JSON. datawave (dw) format is 
canonical(generical) format , which acts as a common structure for all transformations.
{
canonical: a standard or common data format used as an intermediate structure to ensure consistent 
transformation between different data formats and systems.
functional-programming: a way of programming that uses functions as the main building blocks and keeps data 
unchanged. 
}
immutability: data cannot be changed after it is created; instead, new data is created for any 
modification. 

for example: 
Mutable (changing data):
x = 10
x = x + 5   // x becomes 15 (changed)

Immutable (not changing original):
x = 10
y = x + 5   // x is still 10, y is 15 (new value created)

Objects are enclosed in curly braces {} and store data as key-value pairs, while arrays are enclosed in square 
brackets [] and store a collection of values. In DataWeave, scope is created using the do statement, which 
allows defining variables and 'functions within a limited context'.

Within a limited context means variables and functions defined inside a scope (like a do block) are accessible 
only within that block and not outside it.

%dw 2.0
output application/json
---
do {
  var x = 10
  ---
  x + 5
}

- x is only available inside the do block.
- we cannot use x outside this block.

A lambda function in DataWeave is an anonymous function that can be assigned to variables or passed as an 
argument. It is commonly used in higher-order functions like map and filter. Shorthand variables like $, $$ 
and $$$ represent value, index/key and additional parameters depending on the context.

$, $$, $$$ are shorthand variables used in lambda functions:
	$   -> value (first parameter)
	$$  -> index or key (second parameter)
	$$$ -> third parameter (rarely used)

When there is no key (like in arrays):
	$  represents value
	$$ represents index

When working with objects:
 $  represents value
 $$ represents key

A function without a name is called a Lambda function, lambda is a value (like a string or number), so it 
Lambda functions are useful when passing a function as an argument or returning a function from another 
function (Higher-Order Function).
Anonymous (lambda) functions are unnamed functions used inline and higher-order functions are functions that 
take or return other functions. In DataWeave, lambda functions are commonly used with higher-order functions 
like map, filter, and reduce.
can be assigned to variables. syntax: (arguments) -> body
A concise anonymous function using arrow syntax --> in dataWeave, all anonymous functions are lambda functions.
[1, 2, 3] map ((x) -> x + 1) Output: [2, 3, 4]

Higher-Order Function:
A Higher-Order function is a function that either accepts another function as an argument or returns a function
as output. It enables writing flexible and reusable logic. In dataWeave, functions like map, filter and reduce 
are common examples. In simple terms, when a function is used within another function (passed or returned), it 
is called a higher-order function.

%dw 2.0
output application/json
fun applyTwice(f, x) = f(f(x))
var square = (n) -> n * n
---
applyTwice(square, 2)  // Output: 16

%dw 2.0 
output application/json
fun applyTwice(f, x) = f(f(x))
fun increment(n) = n + 1
---
applyTwice(increment, 3) // returns 5

In DataWeave, Prefix_notation means the function name is written before its arguments, like 
functionName(argument). 
On the other hand, Infix_notation is used when a function takes two arguments, where the function name is 
written between the arguments and often a lambda function is used as the second argument, such as 
(argument1) functionName ((argument2) -> expression).

'Variables are two types pre-defined variables and user defined variables':

1. predefined variables(The scope of variables are application level):
These predefined variables and functions provide a powerful set of tools for performing data transformations
and manupulations within DataWeave.
1. message
2. attributes
3. payload
4. vars
5. error
6. app 
7. flow
8. mule 
9. server
Payload, attributes, vars and error are not traditional variables; they are part of the mule-event structure. 
However, in dataWeave they are treated as predefined references that can be directly accessed for transformation, 
which is why they are often referred to as predefined variables.
The payload represents the main input-data being processed. The attributes object contains metadata such as 
headers or query parameters. the vars object is used to store and access flow variables, including commonly 
used values like a correlation ID for tracking. The error object provides details about any error that occurs 
during processing. The server object gives information about server configurations like host and timezone, 
while the flow object contains details about the current Mule flow. The message object represents the complete 
message, including both payload and attributes. additionally, functions like now() return the current date and 
time, p() is a shortcut for accessing properties, and sizeOf() is used to determine the size or length of data 
structures like arrays or objects.

%dw 2.0
output application/json
---
{
"app": app.encoding ,
"flow": flow.name ,
"mule": mule.version ,
"server": server.timeZone 
}

%dw 2.0
output application/json
---
{
	"app": {
		"namespace": app.encoding,
		"applicationName": app.name
	}
}

2. user-defined variables:
These variables will be created by us in our mule flow , their are two types mainly:
	1. Flow level variables(The scope of variable is flow level). 
	2. Datawave variables(The scope of variable is datawave-level).

1. Flow-level variables: are user-defined variables whose scope is limited to the mule-flow. they can be created 
using components like set-variable or by using the Target option in components such as Transform Message. these 
variables store values that can be reused across different processors within the same flow.

2. Datawave variables:
typical DataWeave script has two main parts:
2.1 Header Section (Above the --- line)
	- This is where we define:
	- Global variables
	- Functions
	- Imports
	- Namespaces
2.2 Body Section (Below the --- line)
	- This is where we apply the main transformation logic.
	- we can define local variables here using var, but their scope is limited to the block or expression 
	they are defined in.

In DataWeave, by importing dw::core::Arrays, we can use various predefined array functions to manipulate data 
efficiently.

1. countBy   - Counts elements in an array based on a given condition.
2. divideBy  - Divides an array into smaller arrays of a specified size.
3. indexOf   - Returns the index of a specified element in the array.
4. slice     - Extracts a portion of an array between given start and end indexes.
5. sumby     - Calculates the sum of elements based on a given expression.
6. splitAt   - Splits an array into two parts at a specified index.
7. dropWhile - Removes elements from the beginning of an array while a condition is true.
8. join      - Combines all elements of an array into a single string using a separator.


countBy:  is a higher-order function that groups elements of an array based on a condition or key and then 
counts the number of elements in each group. It returns the result as an object, where the keys represent the 
grouping criteria and the values represent the count of elements in each group.


divideBy: The divideBy function splits an array into smaller chunks of a specified size and returns the result 
as an 'array of arrays'.  
[ ] --> [[] , []]

indexOf: is used to find the index of the 'first-occurrence' of a specified value in an array. it returns the 
position of that value and if the value is not found, it returns -1.
example: indexOf(var_name , "value")

slice: is used to extract a portion of an array between a start index and an end index. it returns a new array 
containing only the selected elements. The syntax is slice(array, fromIndex, untilIndex), where the fromIndex 
is inclusive and the untilIndex is exclusive.

sumby: it works on array and returns the sum of the values of the elements. sumBy is used to calculate the sum 
of values in an array based on a given expression or key. it can be used with arrays of objects or simple arrays 
by applying the appropriate expression.
example: 
1. Array of objects: [{price: 10}, {price: 20}] sumBy $.price  //[] sumBy $.keyName.
2. Array of numbers: [1, 2, 3, 4] sumBy $

splitAt: is used to split an array into two parts at a specified index. it returns an array containing two 
sub-arrays, where the first part contains elements before the index and the second part contains the remaining 
elements.

dropWhile: is used to remove elements from the beginning of an array as long as a given condition is true. Once 
an element does not satisfy the condition, it stops dropping and returns the remaining elements of the array.

join: is used to combine elements of an array into a single string using a specified separator. it takes an 
array and joins all its values into one string.

read: is used to parse a string or binary data into a structured DataWeave format such as an object or 
array. It is mainly used when the content type cannot be automatically detected, so we explicitly provide the 
format like JSON, XML, or CSV. this helps convert raw input data into a usable structure for transformation. 
example: 
%dw 2.0
output application/json
var csvData = "name,age\nJohn,30\nSam,25"
---
read(csvData, "application/csv")

readUrl: is used to read and parse data from an external file or resource (such as classpath, file system, or 
URL) by specifying the file path and content type, allowing external data to be used within a dataWeave 
transformation.
readUrl("classpath://sayHello/uri_input_attributes-success.json", "application/json")

Date & Time: are used to represent and handle different forms of date and time values in transformations. 
'Date' represents only year, month and day (e.g., 2012-10-11).  'DateTime' includes date, time and timezone 
information (e.g., 2012-10-11T10:10:10-03:00).  'LocalTime' represents only time values without date 
(e.g., 10:10:10).'LocalDateTime' includes both date and time without timezone. 'Time' value represents time 
along with timezone offset (e.g., |05:00|). 'Timezone' indicates the offset from UTC, such as +05:00.

Additionally, 'DatePeriod' (e.g., P1Y, P3M, P2Y6M5D) is used to represent calendar-based durations like years, 
months and days, while 'Duration' (e.g., PT1H, PT30M, PT1H20M15S) is used for time-based durations like hours, 
minutes and seconds. These types help in performing accurate date and time calculations in dataWeave.

CDATA stands for Character Data. It is used in XML to include text data that should not be treated as XML 
markup by the parser. anything written inside a CDATA section is treated as plain text, even if it contains 
special characters like < > or & which would normally be interpreted as XML tags.

scan: is used to search a string and return all matching patterns as an array based on a regular expression. it 
extracts all occurrences that match the given regex and returns them in a structured format.
example:
%dw 2.0 output application/json ---
"www.mathoman.com" scan(/([a-z]*)\.([a-z]*)\.([a-z]*)/)

zip: is used to combine two arrays element by element into a 'single-array of pairs'(array of arrays). 
each pair contains elements from the same index of both arrays. it stops when the shortest array ends.

unzip:  is the opposite of zip; it splits an array of paired values back into two separate arrays.
example: %dw 2.0 output application/json ---
// zip([1,2,3,4], ["a","b","c"])
// unzip([[1,"a"], [2,"b"], [3,"c"]])

sizeOf: is used to return the size or length of a data structure. if the input is a string, it returns the 
number of characters including spaces. if the input is an array or array of arrays, it returns the number of 
elements, and if the array is empty, it returns 0. It can also be used with objects to get the number of 
key-value pairs. for example, sizeOf(payload.items) returns the count of elements in the items array.

then is used as a conditional chaining function that works like a pipe operator for non-null values. It takes 
the result of the previous expression and passes it to the next function (callback) only if the value is not 
null. If the value is null, it stops execution and does not pass anything forward. this helps in safely 
chaining transformations without causing null-related errors.

wait: is used to pause the execution for a specified time in milliseconds. When this function is called, it 
blocks the current thread until the timeout period is completed before continuing execution. However, it is 
important to note that using wait can negatively impact performance because it blocks the runtime thread, 
potentially causing slowness or even freezing the application. Therefore, it is intended only for limited 
functional testing purposes and should be 'avoided in production or performance-critical applications'.
------------------------------------------------------------------------------------
lookup: is a built-in function used to invoke a private flow from within a DataWeave script and retrieve its 
output for further transformation. It allows us to pass input parameters to the flow and use the returned 
result in the DataWeave expression. The syntax is Mule::lookup('flowName', payload). It is important to note 
that lookup() works only with private flows, because they are independently executable, whereas subflows can't 
be called using lookup() since they are not standalone execution units.
