Map: is used to iterate over an array and transform each element, returning a new array as output. It works on 
arrays such as simple arrays [], arrays of objects [{}]and even nested arrays[[]]. The map function takes an 
input-array and applies a lambda (transform) expression to each element. The result of each iteration is 
collected into a new array.
Inside the lambda expression:
  - $ represents the current element.
  - $$ represents the index-position of the element.

For-example: in an array of objects[{}], $ refers to the object itself while $$ gives the index of that object 
in the array. the general syntax is 'array map (item, index) -> transformation', where each element is processed 
individually and returned as a transformed array. In muleSoft anypoint-platform, map is one of the most commonly 
used functions for data transformation, restructuring payloads and converting input-data into required output 
formats.
It works like a for-each loop over an array. it iterates through each element, temporarily assigns it to the
variable $, applies the transformation logic and returns a new transformed array. It doesn't modify the original 
payload; instead, it produces a new output array based on the transformation-logic.

syntax-1: Without explicit parameters (shorthand form)
arrayPayload map{
($$):$
}
syntax-2: with parameters (explicit form)
arrayPayload map(value,index)->{
(index):value
}
------------------------------------------------------------------
mapObject: is used to transform an object by iterating over each key-value pair and returning a new object as 
output. Unlike map, which works on arrays, mapObject works on objects {} and nested objects {{}}, where each 
property is processed individually. it takes two inputs: the object to transform on-left-side and a lambda 
expression on-right-side that defines the transformation logic. during iteration, $ represents the value, $$ 
represents the key and mapObject is commonly used when we need to rename keys, restructure JSON objects or 
transform object-based payloads into a different schema while preserving object format.

syntax-1: Without explicit parameters (shorthand form)
objectPayload mapObject {
    ($$): ($)
}}
syntax-2: with parameters (explicit form)
objectPayload mapObject ((value, key, index) -> {
    (key): value
})
is used to iterate over object key-value pairs and transform them while preserving object structure. in 
shorthand form, we use $ for value and $$ for key. In explicit form, we define parameters like value and key in 
a lambda expression. The output remains an object.

Filter-function: is used to iterate over an array and return only those elements that satisfy a given 
condition. It evaluates each element of the array using a lambda expression that must return a boolean value 
(true or false). If the condition evaluates to true, the element is included in the output; otherwise, it is 
excluded. the filter function works only on arrays ([]) and returns an array as output, containing only the 
matching elements. It's commonly used in transformations to perform data selection or to extract specific 
records from a dataset based on conditions like age, status or Id matching.

syntax-1: Without explicit parameters (shorthand form)
  arrayPayload filter(condition)
  $ represents the current value on each iteration.
syntax-2: with parameters (explicit form)
  arrayPayload filter(value,index)-> condition

Filter can be written in two ways. In shorthand form, it uses $ to represent the current element, such as array 
filter ($ > 10). In explicit lambda form, we define parameters like value and index using 
(value, index) -> condition. both forms evaluate a boolean condition and return a new filtered array.

{
Implicit means dataWeave automatically provides parameters like $ without explicit declaration, making the 
syntax shorter. explicit means we manually define parameters like value and index in lambda expressions, giving 
more control and clarity in transformations.
}
----------------------------------------
filter-object: is used to iterate over key-value pairs of an object and return only those pairs that satisfy a 
given condition. It evaluates each entry using a lambda expression that must return a boolean (true or false). 
if the condition is true, the key-value pair is included in the output object; otherwise, it is excluded.
Unlike filter, which works on arrays, filterObject works on objects {} and returns a filtered object as output, 
preserving object structure while removing non-matching entries.

syntax-1: Without explicit parameters (shorthand form)
   objectPayload filterObject ((value) -> value > 10)
syntax-2: with parameters (explicit form)
    objectPayload filterObject ((value, key, index) -> value > 10)
we should use explicit variables like value and key instead of $, because $ is primarily meant for array 
operations. Since filterObject works on key-value pairs of objects, using value and key makes the 
transformation clearer and more reliable..

we can use, map + filter → array operations & mapObject + filterObject → object operations

//Dataweave core flatten & flatMap functions:

flatten: is used to convert an array of arrays (nested arrays) into a single flat array by removing one level of 
nesting. It takes a structure like [ [1,2,3], [4,5] ] and transforms it into [1,2,3,4,5]. In other words, it 
merges all inner arrays into one continuous array. this function is commonly used when dealing with hierarchical 
or grouped data that needs to be simplified for processing or integration.

flatMap: is a function that iterates over each element of an array, applies a transformation, and automatically 
flattens the resulting arrays into a single-level array. it's commonly described as a combination of 
map + flatten, because it first transforms each element and then removes one level of nesting from the result. 
the function operates on arrays and uses $ for the current value and $$ for the index during iteration.

//Difference between flatMap and flatten..?
--> 
flatten is used to convert an array of arrays into a single flat array by removing one level of nesting, 
without performing any transformation. flatMap, on the other hand, first applies a transformation to each 
element of the array and then flattens the result into a single array. So flatten only handles structure, while 
flatMap handles both transformation and flattening.

syntax-1: Without explicit parameters (shorthand form)
arrayOfArrayPayload flatMap ($)
where, $ = current value ; $$ = current index

syntax-2: with parameters (explicit form)
arrayOfArrayPayload flatMap ((value, index) -> value)

                    %dw 2.0 output application/json ---
                    [[1,2], [3,4]] flatMap {
                        $  // returns inner arrays
                    }

//distinctBy: is used to iterate over an array and return only unique elements based on a specified condition 
or key. It helps in removing duplicate values from an array by comparing elements using a transformation 
expression. Unlike simple uniqueness checks, distinctBy allows us to define how uniqueness should be determined, 
for example by a specific field in an object or a computed value. It is commonly used when working with arrays 
of objects to eliminate duplicate records based on a business key like id or email.

syntax-1: arrayPayload distinctBy $
syntax-2: arrayPayload distinctBy(value)-> value

orderBy: is used to sort or reorder elements of an array based on a specified sorting condition or key. it 
iterates over the input array and evaluates each element using a lambda expression to determine the sorting 
criteria, such as numeric value, string value or a specific field in an object. the result is a new sorted array, 
either in ascending or descending order depending on the logic provided. this function is commonly used to 
organize data such as sorting records by name, age, date during transformation.

syntax-1: arrayPayload orderBy $
syntax-2: arrayPaylaod orderBy(value)-> value

groupBy: is used to group elements of an array into an object based on a specified condition or key. it 
evaluates each element using a lambda expression and groups all elements with the same result under a single-key, 
returning an object of grouped arrays. [] --> {[]}

syntax: arrayPayload groupBy(value)-> value

joinBy: is used to merge all elements of an array into a single string, using a specified separator between 
each element. during this process, all elements in the array are automatically converted to strings before 
concatenation. it's commonly used for generating formatted outputs such as CSV lines, readable messages or 
custom string formats. The joinBy function performs the opposite operation of splitBy, where splitBy divides a 
string into an array, while joinBy combines an array into a string. 
syntax: arrayPayload joinBy "separator" joinBy ", " joinBy " - "

//reduce function: is used to iterate over an array and produce a single accumulated result by applying a 
lambda expression on each element. It uses an accumulator to store intermediate results and can return different 
types such as number, string, array or object depending on the logic. It is mainly used for aggregation 
operations like sum, concatenation or building complex structures.

syntax-1: array reduce ((accumulator, item) -> expression)
example: [2,3,4]reduce($+$$)

syntax-2: array reduce ((accumulator, item, index) -> expression)
example:[2,3,4] reduce(item,acc)-> item + acc

Explanation:
iteration-1: item is 2 (assigns first item to acc = 2)..
iteration-2: item is 3 & acc is 2 -> 3 + 2 (assigns result to acc = 5)..
iteration-3: item is 4 & acc is 5 -> 4 + 5 (assigns result to acc = 9)..
result:9

                      %dw 2.0 output application/json var numberArray = [2,3,4] ---
                      {
                        "a": numberArray reduce($+$$),
                        "b": numberArray reduce(i, a) -> (i + a)
                      }

Pluck: is used to iterate over key-value pairs of an object and return the result as an array. it allows 
extraction of values, keys or both using a lambda expression. in shorthand, $ represents value and $$ represents
key, while index can be accessed in explicit-form.  

{

'to operatior':  is used to create a range of values between two specified boundaries. it generates a sequence 
including both the start and end values and is commonly used for iteration or generating arrays of numbers.
to = generate sequence from start to end

uuid: it generates a version-4 UUID using random numbers as the source. it produces a unique identifier string 
that is commonly used for tracking, correlation IDs, and ensuring uniqueness in mule integrations.

zip: is used to combine two arrays by pairing elements at the same index and returning them as sub-arrays. it 
produces an array of arrays and stops at the shortest input array-length.

unzip: is used to reverse the zip operation. it takes an array of paired elements as input and splits them into 
multiple arrays based on their positions, effectively separating combined data into individual arrays.

round: is used to round a number to the nearest integer or specified decimal places based on standard rounding 
rules. It does not simply remove decimals but adjusts the value depending on whether the fractional part is 
greater or less than 0.5
